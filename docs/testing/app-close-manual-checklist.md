# 应用关闭回归 — 手工验收清单

自动化测试覆盖 L1 保存策略、`app_close` 不入队版本、以及 `setAppClosing` 调度器门控。以下步骤用于验证 Tauri 窗口关闭与进程退出（无法在无头 CI 中替代）。

## 前置条件

- 已构建或可运行开发环境：`npm run tauri dev`
- 已打开至少一个库（vault），并打开 **2 个以上** 笔记标签（其中一个有未保存编辑）

## 步骤

1. 在标签 A 中输入若干字符，等待自动保存或手动保存，确认状态栏/标题无「未保存」异常。
2. 切换到标签 B，同样做少量编辑。
3. 点击窗口右上角 **关闭**（或标题栏关闭按钮），不要用任务管理器强杀。
4. 观察：
   - 应用在数秒内退出，无长时间卡死。
   - **不出现** `CrashSender.exe` 或 Windows 错误报告弹窗。
5. 重新启动 `npm run tauri dev`，打开刚才编辑的笔记，确认 `.md` 内容已持久化。

## 失败时记录

- OS 版本、是否 Debug/Release 构建
- 打开标签数量、单篇笔记大致字符数
- 关闭时是否弹出保存错误提示（`关闭前保存失败`）
- 如有崩溃，保留 `%LOCALAPPDATA%` 下相关 dump 路径（勿含笔记正文）
