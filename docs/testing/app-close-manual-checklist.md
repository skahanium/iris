# 应用关闭回归 — 手工验收清单

自动化测试不能替代本清单：这里验证 Tauri 窗口关闭、持久化屏障和真实磁盘内容。完整自动化与 Windows E2E 要求见 [文档持久化与嵌入验收](./document-persistence-embedding-acceptance.md)。

## 前置条件

- 已构建或可运行开发环境：`npm run tauri dev`
- 已打开至少一个库（vault），并打开 **2 个以上** 笔记标签（均有未确认落盘的编辑）
- 准备一个全新空白笔记，以便核对标题重命名后的原始 `.md` 文件。

## 步骤

1. 在新笔记中输入唯一标题和至少两段唯一正文；修改标题以触发文件名同步重命名，并在重命名或编辑器重挂载期间继续输入最后一段正文。
2. 不等待防抖完成，立即触发手动保存后点击窗口右上角 **关闭**（或标题栏关闭按钮）；不要用任务管理器强杀。
3. 观察：
   - 应用在数秒内退出，无长时间卡死。
   - **不出现** `CrashSender.exe` 或 Windows 错误报告弹窗。
   - 窗口若显示 `正在保存`，退出只能在所有 dirty 修订得到回执后继续。
4. 重新启动 `npm run tauri dev`，直接读取重命名后的 `.md` 文件并在 Iris 中打开：标题和每一段正文必须逐字一致，不能依赖 SQLite 标题或编辑器缓存判断通过。
5. 单独执行失败路径：在可控的写入失败条件下再次关闭，确认窗口仍保持打开、编辑状态仍在，并出现明确的“保存失败”及“重试 / 返回编辑”操作；没有可信快照时同样不得退出。
6. 在有 dirty 标签时安装已下载更新，确认安装调用同一保存屏障；全部落盘后才安装，失败时不退出也不安装。

## macOS 首次升级：安全手工迁移与验收

适用于从 v1.2.12 或 v1.2.13 首次升级到 v1.2.14。旧版运行时目录位于应用包内，不能依赖其应用内更新功能完成本次迁移；完成本节后，后续版本应可使用应用内更新。

1. 完全退出 Iris，确认没有残留的 Iris 进程。
2. 在 Finder 中将现有 `~/Library/Application Support/com.iris.notes` **整体移走并重命名为备份目录**；如该目录存在，保留这份备份以便恢复。确认原路径此时不存在，**不要**将旧数据与其合并覆盖。
3. 在 Finder 中将旧应用的 `/Applications/Iris.app/Contents/MacOS/.iris` **整体复制到** `~/Library/Application Support`，再将复制出的 `.iris` 目录重命名为 `com.iris.notes`。完成后目标目录的直接子项应为 `app-data`、`cache`、`tmp` 和 `skills`，不得出现额外嵌套的 `.iris` 目录。
4. 从 v1.2.14 DMG 将 Iris 替换到 `/Applications`，再运行 `codesign --verify --deep --strict /Applications/Iris.app`；命令成功后启动应用。
5. 验证以下状态均被继承：LLM/MCP 配置及凭据状态、人格设置、Vault 文件夹选择、Agent 会话历史，以及应用数据库和缓存状态。仅验证状态是否可用；不得在截图、测试记录、日志或缺陷描述中记录或暴露 API Key、其他密钥、Token 或笔记内容。
6. 确认 Markdown vault 文件未被本迁移修改；如需核对，只比较文件数量、修改时间或内容哈希，不在验收记录中抄录笔记正文。
7. 从 v1.2.14 升级到一个后续测试版本，完成一次应用内更新；再次确认步骤 5 的状态不变，并运行同一条 `codesign` 校验命令。

迁移失败时，先退出 Iris，将失败的 `~/Library/Application Support/com.iris.notes` 整体移走，再将步骤 2 的备份目录重命名回 `com.iris.notes`；不要通过合并目录尝试修复。

## 失败时记录

- OS 版本、是否 Debug/Release 构建
- 打开标签数量、单篇笔记大致字符数
- 保存状态从 `正在保存` 到 `已保存`、`保存失败` 或 `已保存但索引待修复` 的实际表现（勿记录笔记标题、正文或路径）
- 是否在保存失败或无可信快照时保持窗口与编辑状态
- 如有崩溃，保留 `%LOCALAPPDATA%` 下相关 dump 路径（勿含笔记正文）
- macOS 首次升级时，仅记录是否完成备份与状态继承验收；勿记录 API Key、Token、加密凭据或笔记内容。
