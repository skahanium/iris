# Iris Rail Refresh Manual Checklist

## macOS 顶栏与右侧窗口控制

- [ ] 左侧 Iris 品牌轨道始终可见，且不可点击（无 Home 跳转）。
- [ ] 右侧最小化、最大化、关闭按钮可点击，拖拽区域不吞点击。
- [ ] 全屏进入和退出后顶栏不重叠。

## Rail Segments Tab

- [ ] 激活、未激活、hover、focus、dirty、close button 状态清楚。
- [ ] 长标题截断稳定，不挤压品牌轨道和窗口控制。
- [ ] 零 Tab 空主面不伪装为普通文档 tab。

## Outline Ghost Spine 长文

- [ ] 收起态仅见左缘细竖线与图标，正文无多余留白。
- [ ] 展开态为透明文字索引列（约 12rem），无边框、无毛玻璃、无浮动卡片感。
- [ ] H1/H2/H3 通过缩进、字重和字号区分，长标题单行截断稳定。
- [ ] 50+ 条目滚动顺滑，列表项按文档顺序排列，不出现刻度重叠。
- [ ] 当前章节鼠尾草绿 2px marker 随光标变化，选中项和普通项区分清楚。
- [ ] ↑/↓、Enter、Esc 键盘导航可用，Enter 跳转到对应章节。
- [ ] 空目录无文案（不显示「暂无章节」），仅保留 ghost spine 几何。

## AI 协作侧车长对话

- [ ] 用户消息、助手消息、处理过程、确认与降级状态层级清楚。
- [ ] Composer 在底部稳定可用；AI 活动可投影到 StatusBar。
- [ ] 工具确认和规则确认与 AI sidecar 视觉一致。
- [ ] 过程区在最终正文开始流式后自动折叠，完成摘要为「答复完毕」。

## 壳层边框与字号（Segment 4）

- [ ] 顶栏、底栏、AI 侧车头部分隔线与任务舱 Overlay 顶栏边框视觉一系（`border-subtle`）。
- [ ] 底栏与次要标注使用 `text-caption`，亮/暗主题下对比可读。
- [ ] Rail Tab 激活与 Outline 当前章节 marker 仍为品牌绿，ghost 几何与正文留白未变。

## 正文与亮色对比（Segment 3）

- [ ] 编辑区行宽与两端对齐稳定；外链与 wiki 链接色分工清楚。
- [ ] 亮色主题下代码块与 callout 对比足够，无「灰成一团」感。

## 任务舱 Overlay

- [ ] Quick Open、全库搜索、文件管理、版本、图谱、管理中心等使用统一任务舱 chrome（命令面板 UI 已退役；全局任务入口走 Overlay / 管理中心）。
- [ ] 键盘导航和底部快捷键提示清楚。
- [ ] 空态、结果卡、选中态一致。
