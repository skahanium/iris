# 文档版本与历史追踪 — 产品设计

**状态**：已批准（2026-05-26）  
**范围**：单篇笔记的版本快照、定稿、恢复、时间线 UI；不含内置 Git。  
**相关**：`ARCHITECTURE.md` 版本节、`ROADMAP.md` v0.3、AGENTS.md 数据原则。

---

## 1. 目标

| 目标     | 说明                                                           |
| -------- | -------------------------------------------------------------- |
| 脉络清晰 | 用户能区分「当前正文」「检查点」「定稿」；时间线按日与类型分组 |
| 更安全   | 恢复前有保护快照；危险操作需确认；存储路径与 hash 一致         |
| 低负载   | 写盘与打版分离；空闲/配额/去重；不做全库定时扫描               |
| 用户友好 | 中文文案区分「自动保存」与「版本」；定稿突出；自动备份默认折叠 |

**非目标（v0.3）**：块级 diff 存储、内置 Git、跨笔记合并快照、版本软删除回收站。

---

## 2. 核心模型

### 2.1 双层保存

```
层 1 · 持久化（当前正文）
  触发：编辑防抖（默认 1200ms，设置 500–2000ms）
  动作：仅写 vault/<path>.md
  文案：「已自动保存」— 不称为「版本」

层 2 · 版本快照（历史 / 定稿）
  触发：见 §3
  存储：.iris/versions/<file_id>/<version_no>.md
  元数据：SQLite `versions` 表
  文案：「版本」「定稿」「恢复前备份」等
```

**铁律**

- 唯一可编辑对象 = 当前 `.md`。
- 快照只读；禁止直接编辑历史正文；回溯仅通过「恢复」覆盖当前版。
- `.md` 为权威来源；`.iris/versions` 为衍生缓存（删除后无法恢复已删快照正文）。

### 2.2 命名与展示

- 新建文档：`新建文档`、`新建文档（1）`… → `files.title`；磁盘文件名与 title 对齐（实现细节见实现计划）。
- 列表、标签栏、最近笔记：**仅显示 `files.title`**；`path` 仅 tooltip / 高级信息。
- 历史/定稿**不出现在**文档列表中。

### 2.3 与 Git 的关系（产品叙事）

帮助/设置中说明：

- **当前文件**：自动保存到笔记 `.md`。
- **版本历史**：Iris 为本篇保留的检查点。
- **整库历史**：用户可在 vault 目录自行使用 Git（ROADMAP「Vault 外工具链」）。

## 3. 快照类型 `kind`

新增列 `kind TEXT NOT NULL`（枚举字符串）：

| kind          | 触发条件                                                 |
| ------------- | -------------------------------------------------------- |
| `auto_idle`   | 该文档连续无编辑 ≥ N 分钟（默认 10）且 content_hash 变化 |
| `manual`      | Ctrl+S 或菜单「保存版本」                                |
| `pre_restore` | 执行恢复前自动创建                                       |
| `finalize`    | 定稿：对**当前内容**新建快照并 `is_finalized=1`          |
| `pre_close`   | 关闭该文档标签或应用退出前（P2，可选）                   |

定稿操作：**不再**仅 UPDATE 已有行；必须 **INSERT** 新快照 + `is_finalized=1` + 可选 `label`。

## 4. 触发与节流

### 4.1 不再触发的行为

- ~~每次 `file_write` 自动 `create_snapshot`~~（与层 1 解耦）。

### 4.2 快照规则

| 规则                | 值                                                      |
| ------------------- | ------------------------------------------------------- |
| 同 content_hash     | 跳过插入（保留）                                        |
| 空闲快照最小间隔    | 同一文档 ≥ 10 分钟                                      |
| 连续 auto_idle 防抖 | 距上一条任意快照 < 2 分钟且拟插入 kind=auto_idle → 跳过 |
| 单篇 auto_idle 上限 | 30 条；超出删除最旧非定稿 auto_idle                     |
| 非定稿全局清理      | 创建后 7 天删除（保留）                                 |
| 定稿                | 永不自动删除                                            |

### 4.3 设置档位（P1）

| 档位         | 空闲间隔       | auto_idle 上限 |
| ------------ | -------------- | -------------- |
| 仅手动版本   | 不打 auto_idle | 0              |
| 标准（默认） | 10 分钟        | 30             |
| 更勤备份     | 5 分钟         | 50             |

## 5. 时间线 UI

### 5.1 结构

- 按日分组：今天 / 昨天 / 更早。
- **定稿**条目：置顶或独立「定稿」区，角标 + `label`。
- **手动 / 恢复前 / 关闭前**：平铺于对应日期组。
- **自动备份（已批准 A）**：默认 **折叠** 为「自动备份（N）」，展开后列出各 `auto_idle` 条目。
- `pre_restore` 建议归入折叠组「恢复相关（N）」或并入自动备份折叠策略（实现时二选一，默认与 auto 分开展示更清晰）。

### 5.2 对比与恢复

- 双栏：左（上）当前版摘要；右（下）选中快照只读。
- 恢复文案：「将当前正文替换为此版本」；说明将创建恢复前备份。
- 确认：当前有未 flush 编辑 **或** 目标为定稿 → 二次确认。

### 5.3 定稿删除

- 定稿可删除，**强确认**（不可恢复）。
- 定稿快照正文不可改；仅允许修改 `label`（可选）。

## 6. 安全与一致性

| 机制           | 说明                                             |
| -------------- | ------------------------------------------------ |
| `storage_path` | 固定 `<file_id>/<version_no>.md`，与磁盘路径一致 |
| 读快照         | 校验文件存在；可选校验 hash                      |
| 恢复前         | 强制 `pre_restore`                               |
| 退出/关标签    | 层 1 flush；满足条件时 `pre_close`（P2）         |
| 外部文件变更   | 走 L1/L2/L3 冲突流后再写盘/提示是否打版          |

---

## 7. 数据 schema 变更（概要）

`versions` 表增加：

```sql
kind TEXT NOT NULL DEFAULT 'manual'
```

迁移需提供 `down`。既有行回填为 `manual` 或 `auto_idle`（按 `is_finalized` 推断）。

修复既有 `storage_path` 错误数据（迁移或一次性修复脚本）。

## 8. 实施分期

| 阶段   | 内容                                                            |
| ------ | --------------------------------------------------------------- |
| **P0** | 双层分离；`kind` + 触发重写；`storage_path` 修复；定稿=新建快照 |
| **P1** | 折叠时间线；双栏对比；Ctrl+S；配额；设置档位                    |
| **P2** | `pre_close`；冲突流联动；占用统计                               |

## 9. 验收标准

- [ ] 连续输入仅更新 `.md`，不在 2 分钟内刷 auto_idle 版本条。
- [ ] Ctrl+S 产生 `manual` 版本（hash 变时）。
- [ ] 定稿产生新 `finalize` 行且 7 天清理不删。
- [ ] 版本面板默认见折叠「自动备份（N）」，展开可见条目。
- [ ] 恢复前存在 `pre_restore`；恢复后当前正文与目标一致。
- [ ] `version_preview` / `version_restore` 路径正确可读。
