# 产品合同与交互原则

## 1. 目标

Iris Agent 的价值是降低复杂工作的表达和分析成本，而不是让用户管理一个 Agent 平台。默认对话面板必须保持“自然语言优先、内部能力无感、真实副作用可控”。

用户日常只需要：

1. 说出目标或继续讨论。
2. 必要时使用 `@` 指定资料。
3. 在写回前确认一份可读的变更预览。

不得要求普通用户选择 scene、workflow、Skill、MCP server、子 Agent 数量、证据策略或模型故障转移方式。

## 2. Session 与编辑器硬解绑

以下是产品和数据结构共同保证的不变量：

- 新建或继续 Agent Session 不记录当前文档路径。
- 打开、切换、重命名或关闭编辑器文档，不改变 Session 上下文。
- Agent 发送按钮不得隐式附带当前文档、选区、同目录文件或当前 tab 信息。
- Session 标题和历史列表不得按当前文档自动分组。
- 文档只能通过本条消息的显式 `@` 引用或显式编辑器 AI 动作进入 Run。
- 编辑器 AI 动作携带的是本次 Run 的目标与快照，不创建后续 Session 绑定。
- 后续“继续修改刚才那篇”只能解析对话中已有的明确引用，不能读取当前编辑器状态补全。

## 3. 核心用户旅程

### 3.1 普通问答

- 简单、稳定且无需外部事实的问题直接流式回答。
- 联网开启时，对时效性或外部可核验事实主动搜索。
- 不因出现某个关键词就进入写作、研究或组织工作流。
- 不显示内部规划步骤；仅显示简短状态和必要来源。

### 3.2 公文写作

- 用户可以在纯对话中提出写稿目标，也可以显式 `@` 现有材料。
- Harness 自动优先寻找相近文种的范文样本，学习结构、层次、语气和表达。
- 范文中的人名、事实、数字和结论不得复制为新稿事实。
- 如果内容涉及工作规则、政策边界或合规判断，再同时使用规范依据约束内容。
- 初稿先出现在对话或临时产物 tab；写入 `.md` 前展示完整变更计划。

### 3.3 工作分析

- Agent 应通过多轮对话理解目标、约束、利益相关方、事实缺口和风险。
- 与工作相关的判断必须主动重视规范依据，不能只靠一般常识给结论。
- 输出要区分：已知事实、规范要求、推断、风险、备选方案和建议。
- 对高影响假设、规范冲突和不可逆选择必须回到用户，而不是擅自补全。
- 用户要求把讨论形成公文时，内容由规范依据约束，写法由范文样本约束。

### 3.4 小说写作

- Agent 面板不得自动读取当前小说、其他章节、人物卡、设定集或同目录内容。
- 仓库文档只有被用户明确 `@` 后才能进入本轮上下文。
- 用户从编辑器主动发起续写、润色等动作时，仅本次传入的快照可用。
- 没有明确事实需求时不主动联网。
- 不自动应用公文规范、工作资料或文件夹遵循等级。

### 3.5 课题研究

本次重构明确不提供专门的课题研究场景。普通联网查询和工作分析仍可检索证据，但不得出现 Research executor、ResearchState、研究控制面板或“自动研究到底”的产品承诺。

## 4. 默认交互

### 对话中可见

- 当前简短阶段：如“正在查阅规范依据”“正在联网核实”“正在整理草稿”。
- 流式回答、引用和高价值临时产物。
- 必要的澄清问题。
- 一次性的变更预览与确认。
- 可恢复错误及明确下一步。

### 默认隐藏

- 工具名称、权限原子、MCP 映射、Provider 候选和 checkpoint。
- Skill 分值、路由标签和执行预算。
- 内部思维链、候选答案和反思文本。

高级“运行详情”可以展示脱敏后的阶段耗时、工具类别、实际 Provider、Skill 激活原因、证据来源和恢复记录。

## 5. 澄清原则

Agent 只在答案会实质改变时提问：

- 目标或受众缺失会改变成品。
- 高影响假设无法从已授权资料中确定。
- 多份规范依据冲突。
- 写回目标或影响范围不明确。
- 需要的权限、文档或安全域未授权。

普通措辞偏好、可安全推断的格式和可逆的只读探索不应阻塞用户。

## 6. 产品验收不变量

- 切换当前文档后，同一问题的 Harness 输入不发生变化。
- 没有 `@` 或显式编辑器动作时，小说请求的仓库读取次数为零。
- 公文写作对范文样本的结构学习优先于规范依据。
- 工作分析涉及规则判断时，规范依据检索优先于一般参考资料。
- 普通用户完成一次问答不需要进入设置页。
- 未发生真实副作用时不弹权限确认。
- 写回确认展示业务结果和 diff，不展示底层工具参数。
