## 状态口径

| 状态       | 含义                                                   |
| ---------- | ------------------------------------------------------ |
| 成立       | 源码或现有测试能证明问题存在。                         |
| 部分成立   | 方向成立，但审计描述存在范围、严重度或实现细节误差。   |
| 不成立     | 当前源码已经具备对应能力，或审计描述与事实冲突。       |
| 需实验验证 | 需要模拟 LLM、性能压测、跨平台运行或安全实验才能定性。 |

## 冲突裁决

- subagent 并发：MIMO 报告称无并行，DeepSeek 报告称并行。源码中 `run.rs` 当前使用 `join_all` 聚合 subagent futures，因此阶段 0 裁决为“并发读基线存在，但缺 coordinator、资源锁、结构化报告和证据继承”。
- agent 权限评分：MIMO 给出较高分，DeepSeek 指出主线执行未强制 permission atoms。源码显示 `ToolPolicy` 仍是主执行门控，`permission preflight` 主要用于 UI/确认摘要，因此阶段 0 裁决为阶段 2 的结构性缺口。
- 沙箱评分：MIMO 强调应用层沙箱，DeepSeek 强调 OS 级沙箱缺失。阶段 0 裁决为“L0/L1 局部边界存在，L2 OS 级边界未实现，必须诚实分级”。
