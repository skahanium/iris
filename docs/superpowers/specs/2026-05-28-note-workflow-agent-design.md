# Iris 笔记工作流 Agent 长期设计

> 设计日期: 2026-05-28
> 状态: 设计草案
> 范围: AI 检索、联网搜索、笔记工作流 Agent、受控文档编辑、文件标题/路径、文件夹与模板管理

## 一、设计结论

Iris 的 AI 不应继续停留在普通聊天侧栏形态，也不应直接变成长期自治、难以控制的通用 Agent。

更合适的方向是：**手动选择的笔记工作流 Agent**。用户明确选择任务，Iris 执行可追踪的工作流：计划、检索本地材料、可选联网搜索、证据融合、引用校验、生成可确认补丁或整理建议。

这份文档是长期蓝图，不只覆盖第一阶段。实施可以分阶段推进，但后续能力都应回到同一条路线中承接。

核心原则：

- **本地优先**：本地笔记、标题、段落、条款、模板是主要证据来源。
- **Markdown 权威**：`.md` 文件仍是用户知识唯一权威来源。
- **证据优先**：AI 回答和写作建议必须能回查证据包。
- **联网可降级**：MiniMax 主通道失败时降级 DuckDuckGo，仍失败则明确说明网络证据不可用。
- **写入必须确认**：AI 不静默修改 `.md`；所有正文、标题、路径、模板、标签、文件移动都必须预览并确认。
- **可追踪可评测**：每次检索、联网、证据混合、补丁生成和写入确认都有 trace。

## 二、长期能力阶段

### Phase 0：文件与索引地基

目标：先让笔记库的基础数据模型可靠，避免后续检索和 Agent 继续依赖错误的路径语义。

能力：

- 废弃 legacy `untitled-*` 新建路径和展示路径。
- 新文档默认使用 `新建文档` / `新建文档（n）`。
- 当用户首次确认有效标题后，可将文件路径同步为人类可读标题路径。
- 文档展示名、搜索名、引用名以 frontmatter/title/body 索引为准，不依赖路径 stem。
- 路径同步必须处理冲突、打开标签、版本快照、索引刷新和文件 watcher。
- 笔记库面板补齐文件夹新建、重命名、删除。
- 模板管理补齐模板新建、读取、编辑、删除，并支持从当前文件夹使用模板新建笔记。

验收：

- 新建笔记不会产生 `untitled-*` 路径。
- legacy `untitled-*` 不参与展示名和检索排序依据。
- 删除 SQLite 后，文件标题、标签、正文、模板索引可从 `.md` 和 `.iris/templates` 重建。

### Phase 1：可信检索与研究写作 Agent

目标：把 AI 侧栏从普通聊天升级为任务工作台，先把“研究问题、辅助写作、检查引用”做可靠。

侧栏任务：

- `研究问题`：围绕用户问题检索本地材料和可选网络来源，输出证据矩阵、可信度说明和带引用回答。
- `辅助写作`：围绕当前选区或光标邻域生成写作建议和可确认补丁。
- `检查引用`：检查当前段落/选区的依据是否充分，推荐本地条款、笔记或网页来源。

后端路线：

- 新增或强化 Rust `Workflow Runtime`，不要把任务状态塞回普通聊天消息。
- 工作流状态包含任务类型、计划步骤、检索状态、联网授权、证据包、模型输出、补丁候选、错误状态。
- React 只负责任务入口、证据展示、工具轨迹、diff 预览和确认。

验收：

- 联网失败不会中断本地研究任务。
- 回答中引用的证据能在 UI 中回查。
- 写作任务只生成补丁，不自动写入 Markdown。

### Phase 2：整理建库 Agent

目标：让 Iris 能帮助用户维护笔记库结构，但所有变更仍由用户确认。

能力：

- 建议标题、标签、文件夹归类、语料库归属。
- 建议双链、块级链接、相关笔记关系。
- 从范文或已有笔记中提取模板。
- 输出批量变更计划，用户可逐项接受、拒绝或批量确认。

边界：

- 不自动移动文件。
- 不自动改 frontmatter。
- 不自动保存模板或长期规则。

验收：

- 批量计划每一项都有来源、理由和影响范围。
- 所有文件移动、标签写入、模板保存都可撤销或通过版本记录追踪。

### Phase 3：章节级与文档级写作

目标：从选区/光标邻域补丁扩展到更大范围的写作协作。

能力：

- 当前章节级改写、续写、结构重排。
- 整篇文档的大纲检查、引用缺口检查、风格一致性检查。
- 跨文档引用建议和证据补全。
- 更强的 diff 分块、冲突检测、版本快照和回滚体验。

边界：

- 整篇重写默认只给计划和补丁分块，不一次性覆盖全文。
- hash 不匹配、文件已变更、范围无法定位时必须阻止直接应用。

### Phase 4：半自治研究工作流

目标：对复杂研究目标支持有限循环，但仍不是无限自治 Agent。

能力：

- 拆解子问题。
- 多轮本地检索和联网搜索。
- 证据矩阵、观点冲突检测、缺口清单。
- 生成研究笔记或写作提纲。
- 用户可在每轮后暂停、修改方向或终止。

边界：

- 工作流必须有最大轮数、最大联网次数、最大工具调用数。
- 只允许任务级联网授权。
- 不允许后台长期运行和静默写文件。

### Phase 5：个人知识记忆与规则

目标：让 Agent 在用户确认后沉淀个人偏好和项目语境。

能力：

- 保存写作偏好、引用格式、可信来源规则、常用项目背景。
- 保存文种模板和表达风格特征。
- 规则可查看、停用、删除。
- 记忆默认不污染知识检索，只有对应工作流需要时才参与上下文。

边界：

- 长期规则必须用户确认。
- 不保存 API Key、用户敏感正文、加密密码到日志、数据库或明文文件。

## 三、核心架构

```text
React AI 侧栏
  ├─ 任务入口：研究问题 / 辅助写作 / 检查引用
  ├─ 计划与进度
  ├─ 证据包抽屉
  ├─ 工具轨迹
  └─ Patch diff 确认

Rust Workflow Runtime
  ├─ Workflow Planner
  ├─ Local Retrieval Broker
  ├─ WebSearch Broker
  ├─ Evidence Mixer
  ├─ Model Gateway
  ├─ Patch Proposal Builder
  ├─ Guardrails
  └─ Trace / Eval

Local Knowledge
  ├─ Markdown files
  ├─ SQLite cache/index
  ├─ sqlite-vec / FTS
  ├─ templates
  └─ sessions / traces / user-confirmed rules
```

职责划分：

| 层级 | 职责 | 不做 |
| --- | --- | --- |
| React | 任务选择、状态展示、证据卡、diff 确认 | 不绕过 IPC 写文件，不直接保存密钥 |
| Workflow Runtime | 编排计划、检索、联网、证据融合、补丁生成 | 不静默修改 `.md` |
| Retrieval Broker | 本地 FTS/vector/graph/exact/template 检索 | 不判断网页可信度 |
| WebSearch Broker | 联网搜索、降级、标准化、来源分级 | 不把网页当高可信本地事实 |
| Evidence Mixer | 去重、排序、融合、预算裁剪 | 不编造证据 |
| File/Template Commands | 文件夹、模板、路径同步 | 不要求用户删除数据库重建 |

## 四、证据与联网搜索设计

### 4.1 统一证据包

本地和网页材料都应进入统一的 `ContextPacket` 模型。现有字段继续保留：

- `source_type`
- `source_path`
- `title`
- `heading_path`
- `source_span`
- `content_hash`
- `excerpt`
- `retrieval_reason`
- `score`
- `trust_level`
- `citation_label`
- `stale`

网页证据需要扩展或附带元数据：

- `url`
- `domain`
- `published_at`
- `fetched_at`
- `search_backend`: `minimax` / `duckduckgo`
- `source_rank`: `official` / `academic` / `media` / `community` / `unknown`
- `failure_reason`
- `fallback_from`

### 4.2 WebSearch Broker

联网搜索不应直接把搜索摘要拼进用户消息。应改为：

1. 规范化查询，避免空 query 和 `missing query parameter`。
2. 调用 MiniMax Token Plan 搜索。
3. MiniMax 失败时降级 DuckDuckGo。
4. 解析结果为结构化 `WebSearchResult`。
5. 去重 URL、规范化域名。
6. 按来源可信等级标注。
7. 转换为 `ContextPacket`。
8. 把失败、降级、空结果作为状态返回前端。

策略：

- 官方/学术来源优先进入证据。
- 社区/未知来源可作为线索，但回答中必须标注不确定。
- 联网搜索失败不应阻断本地证据回答。
- 问“今天几号”等日期类问题优先使用本机日期，不依赖网页摘要。

### 4.3 Evidence Mixer

`EvidenceMixer` 负责把本地和网页证据融合：

- 同源 URL 或同文件片段去重。
- 同一法规条款优先 exact parser。
- 本地用户笔记优先级高于网页。
- 网页只补充时效性、外部事实和本地缺口。
- 输出给模型的证据必须带引用标签。

## 五、Agent 工作流设计

### 5.1 研究问题

输入：

- 用户问题
- 当前笔记路径，可为空
- `@` 指定范围
- 是否允许联网

流程：

1. 生成研究计划。
2. 本地结构化检索。
3. 如联网已授权，执行网页搜索。
4. 证据融合与可信度标注。
5. 生成证据矩阵。
6. 输出带引用回答。
7. 给出可选后续动作：生成提纲、写入研究笔记、补写当前段落。

### 5.2 辅助写作

输入：

- 当前选区或光标邻域
- 当前文档标题、路径和 hash
- 用户写作目标
- 可选证据包

流程：

1. 判断写作意图：续写、改写、补依据、生成提纲、统一语气。
2. 检索本地证据。
3. 如需要且已授权，检索网页证据。
4. 生成写作建议。
5. 生成 `PatchProposal`。
6. 前端 diff 预览。
7. 用户接受后写入。

第一阶段只允许选区/光标邻域补丁；章节级和整篇文档留到 Phase 3。

### 5.3 检查引用

输入：

- 当前段落/选区
- 当前文档上下文
- 可选本地范围
- 是否允许联网

流程：

1. 抽取段落中的事实陈述和引用需求。
2. 本地查找支持或冲突证据。
3. 可选联网补充外部来源。
4. 输出引用充分性检查。
5. 给出可插入引用或改写建议。

## 六、受控编辑与 PatchProposal

AI 对 Markdown 的所有正文写入都必须走 `PatchProposal`。

建议结构：

```typescript
interface PatchProposal {
  id: string;
  targetPath: string;
  baseContentHash: string;
  range: {
    start: number;
    end: number;
  };
  originalText: string;
  replacementText: string;
  evidencePacketIds: string[];
  riskLevel: "low" | "medium" | "high";
  warnings: string[];
  createdAt: string;
}
```

应用规则：

- `baseContentHash` 不匹配时禁止应用。
- `range` 越界时禁止应用。
- 原文与当前范围不一致时禁止应用。
- 接受补丁前展示 diff。
- 接受补丁后走现有 `file_write` 和版本快照链路。
- 用户可拒绝、复制、再生成。

## 七、文件标题、路径、文件夹与模板

### 7.1 标题与路径

设计目标：

- 用户不再看到或依赖 `untitled-*`。
- 文件系统中的路径尽量人类可读。
- 检索和引用不依赖路径名。

规则：

- 新建文档默认标题为 `新建文档` / `新建文档（n）`。
- 默认路径与标题一致。
- 当用户首次输入有效标题时，可触发路径同步建议。
- 路径同步必须处理冲突，例如 `标题.md` 已存在时建议 `标题（1）.md`。
- 路径同步要更新打开标签、文件列表、索引、版本关联和 watcher 状态。
- 用户可关闭自动路径同步，改为手动重命名。

### 7.2 文件夹管理

笔记库面板补齐：

- 新建文件夹。
- 新建子文件夹。
- 重命名文件夹。
- 删除空文件夹。
- 非空文件夹删除必须二次确认，并说明影响。

文件夹操作应通过 Rust IPC 完成，统一使用 vault path 安全校验。

### 7.3 模板管理

模板存储仍使用 `.iris/templates`。

补齐能力：

- `template_list`
- `template_read`
- `template_save`
- `template_delete`
- `template_create_note`

模板编辑界面应是普通 Markdown 编辑体验，不放进 `components/ui/` 基础组件。

## 八、测试与验收

### Rust

- MiniMax 请求永远带合法 query。
- MiniMax 失败降级 DuckDuckGo。
- DuckDuckGo 空结果返回结构化状态。
- Web result 规范化、去重、来源分级。
- `EvidenceMixer` 本地优先、官方/学术网页优先、低可信来源降级。
- `PatchProposal` hash 不匹配、范围越界、原文不一致时禁止应用。
- 文件夹和模板 IPC 路径安全。
- 标题同步路径冲突处理。

### TypeScript / React

- AI 侧栏展示三类任务入口。
- 工作流状态、证据包、联网失败、降级信息可见。
- Diff 预览支持接受、拒绝、复制、再生成。
- legacy `untitled-*` 不展示。
- 文件夹和模板 CRUD UI 可用。

### E2E

- 研究任务能输出证据矩阵和引用回答。
- 联网失败时本地回答继续完成，并显示网络证据不可用。
- 写作任务生成补丁但不静默写入。
- 接受补丁后文档内容、版本快照、标签栏标题一致。
- 新建文档、改标题、路径同步、检索结果标题一致。

## 九、明确不做

- 不做 Electron、Node 后端、外部向量数据库。
- 不做长期后台自治 Agent。
- 不做静默写 Markdown。
- 不允许联网搜索绕过用户授权。
- 不把网页来源当成本地知识权威。
- 不把 API Key、Token、用户正文写入日志或明文存储。

## 十、开放问题

- 路径同步默认自动建议还是默认自动执行后可撤销，需要在 UI 设计时再定。
- Phase 4 半自治研究的最大轮数、最大联网次数、最大工具次数需要结合性能评测确定。
- 网页来源可信等级的域名规则可以先内置，后续是否允许用户维护白名单/黑名单待定。
- 整理建库 Agent 的批量变更 UI 需要单独做交互设计。
