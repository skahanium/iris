Iris AI 从“聊天框”升级成“笔记工作流 Agent”

这页只看方向：Iris 的 AI 不再只是问答，而是围绕笔记库完成“研究、找证据、写作、生成可确认补丁”。

A

推荐：手动选择工作流 Agent

用户先选择任务类型，例如“研究这个问题”“帮我补写这一节”“根据资料生成提纲”。Iris 显示计划，执行本地检索和可选联网搜索，最后给证据和可确认的文档补丁。

B

保守：聊天框增强

保留现在的 AI 侧栏，只把搜索结果、引用卡、插入/替换按钮做得更好。改动小，但还是容易像普通聊天软件，Iris 的特色不明显。

C

激进：高度自治 Agent

用户只说目标，Iris 自己规划、多轮搜索、整理、写入。看起来像 Codex，但用于笔记会更难控制，也更容易误改 Markdown。

推荐方案的一次任务流程

研究 + 写作 Agent
1. 用户选择任务
研究 / 写作
2. 搜索本地笔记
标题 / 段落 / 条款
3. 可选联网搜索
官方 / 学术优先
4. 合成答案
证据矩阵 / 引用
5. 生成补丁
预览 / 确认写入
文档标题和路径
不依赖 Untitled
文件夹和模板
基础管理完整
工具轨迹和评测
可追踪、可复现

为什么推荐 A

  • 比聊天框更像 Iris 自己的能力。
  • 能把本地搜索和联网搜索统一成“证据包”。
  • AI 改文档前必须给补丁预览，符合 Markdown 权威原则。

需要补的地基

  • 搜索结果必须有来源、可信度、引用标签。
  • 联网搜索要能处理失败、降级、去重和来源分级。
  • 文件标题、路径、文件夹、模板要先稳定下来。
