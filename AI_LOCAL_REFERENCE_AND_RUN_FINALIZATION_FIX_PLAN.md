# Iris 普通域本地引用与运行终态修复

## Summary

修复两条已确认的断链：

- `@文件` 目前只是展示 token，没有成为 Harness 的 `explicitReferences`；现有浏览器侧引用哈希也与后端磁盘哈希算法不兼容。
- 工具成功后的收尾失败被统一覆盖成 `PersistenceFailed`，导致所有问题都显示“运行暂时无法完成”。

同时重做普通域的引用呈现：候选选择后，输入框和用户气泡只显示浅绿色文件/文件夹名称，不显示 `@`、方括号、独立“引用”行或胶囊标签。

## 引用协议与 Harness

- 将普通域提交改为结构化 `AssistantTurnDraft`：纯文本 `message`、`explicitReferences`、`retrievalScope`、`displayMentions` 分开传递；涉密域继续拒绝这些普通域引用字段。
- `@文件`：
  - 选择候选后，前端以可读文件标题替换输入中的 `@查询`；真实相对路径仅保存在注解元数据中。
  - 发送前通过现有只读 `fileSignature` 获取后端一致的内容哈希，生成 `kind: note` 的全文引用；不得再使用浏览器 FNV 哈希。
  - 后端重新读取、校验路径和哈希后才将正文作为单轮材料注入模型；不得让模型以提示文本中的文件名自行调用 `read_note` 代替显式引用。
  - 超过全文材料上限时不截断、不降级为泛化失败：转为该文件范围内的受限检索；索引不可用时提示用户改用段落引用。
- `@文件夹` 与 `#标签`：
  - 不作为全文附件，不读取整个目录；仅成为本轮本地检索的硬边界。
  - 将 `retrievalScope` 作为 Run 的不可变输入持久化，并传给工具调度；有范围但没有全文材料时，运行强制进入受限本地检索阶段。
  - 所有路径、前缀、标签均在后端规范化、去重并拒绝内部/涉密路径。
- 段落/选区引用：
  - 在 Markdown 载入编辑器时建立“ProseMirror 位置 ↔ 已提交 Markdown UTF-8 字节范围”的源投影。
  - 选区仅在完全映射到当前已提交快照时生成 `selection/paragraph` 引用；跨不可映射片段或含未保存改动时明确提示“请先保存后再引用”，绝不发送客户端正文或猜测范围。
  - 编辑器选区发送侧栏与内联 AI 均复用该引用工厂；删除当前无调用方、无路径、无有效哈希的旧引用构造方式。
- 新增增量 migration，在 `session_messages` 保存 `context_scope_json` 与 `display_mentions_json`；旧消息默认空数组，历史仍可正常加载。为 migration 提供 down 脚本。

## 输入框与对话气泡

- 将 `@[path]` 文本 token 替换为“纯可读输入 + 带位置的 mention 注解”：
  - 候选选中后，输入值实际就是文件标题或文件夹名称，没有隐藏的 `@[]` 编码。
  - 在原生 textarea 下增加与文本完全同长度、同字体、同滚动的无障碍隐藏高亮层；真实 textarea 保持编辑、IME、剪贴板与快捷键行为，标注区只以浅绿色显示。
  - 输入前/后、粘贴、删除、IME 组合输入时维护范围：在标注外编辑仅平移范围；触及标注则解除绑定并保留为普通文本，避免路径与显示文本脱节。
- 用户气泡持久化并恢复同一组范围注解；安全标记后在 Markdown 渲染结果中以受控占位符恢复内联浅绿色文本。
- 删除 `ContextScopeChips`、`MentionMetadataRow` 和会话底部的引用胶囊展示；文件、文件夹、段落引用均只以内联文本存在。
- 文件名重复时，候选列表继续显示相对路径作副标题；输入和气泡仅显示用户选择的标题，悬浮说明保留安全的相对路径/类型。
- 新增 `--ai-mention` 前景色 token，浅绿色但不加边框、底色、圆角或图标；同步更新设计系统与 ROADMAP 对应条目。

## 运行失败与会话一致性

- 将工具循环和直答链路的收尾阶段显式分段：流式事件 flush、Web 降级处理、证据合并、最终输出校验、SQLite finalize、事件投递。
- 每一段保留内部失败原因，并映射为安全终态码：
  - 输出为空/超长；
  - 显式引用已变更或范围无效；
  - 证据关联无效；
  - 持久化失败；
  - 事件投递失败。
- 仅真正未知的编排退出才使用 `PersistenceFailed`；禁止外层 `fail_active_with_sink` 覆盖已分类终态。日志只记录 `runId`、阶段和安全码，不记录提示词、正文、路径或模型原始错误。
- 如果 SQLite 已不可写而无法持久化失败事件，向当前前端发送仅内存的安全失败事件，确保 UI 不悬挂；恢复时按现有状态机清理未终结 Run。
- 将用户气泡和 assistant 占位符移动到 `assistantRun.start` 成功受理之后创建，并按 `runId/clientRequestId` 幂等绑定；提交失败不留下用户气泡，终态失败不会重复插入消息。

## Test Plan

- 前端：
  - 输入 `@` 时显示候选；选择文件/文件夹后，输入与用户气泡只显示浅绿色名称，无 `@`、方括号、胶囊或“引用”行。
  - 输入、删除、粘贴、IME、键盘导航、滚动、复制、会话恢复及同名文件候选均保持引用范围正确或安全解除绑定。
  - `@文件` 发送的请求含有效全文引用；`@文件夹/#标签` 仅发送范围；选区在已保存快照中产生精确字节范围，脏编辑器明确拒绝。
  - Run 受理失败、终态失败、重试和双击发送均不会重复用户气泡或遗留空 assistant 槽位。
- Rust：
  - 覆盖全文引用的路径/哈希校验、范围持久化与检索边界；文件夹范围不得泄漏到范围外路径。
  - 覆盖长文档的范围检索回退、索引不可用、选区范围无效及旧会话空字段兼容。
  - 覆盖工具成功后发生输出超限、证据错误、flush 错误、finalize 错误和事件投递错误；每种情况都产生准确安全码、一次终态、无未验证内容落盘。
  - 增加 App 组合与桌面 E2E：对普通文档提问应以显式材料完成分析，且不会调用“根据文本猜路径”的 `read_note` 路径。
- 验收前运行前端 format、lint、typecheck、完整测试、桌面 E2E；Rust fmt、clippy、完整测试与前后端依赖审计。

## Assumptions

- `@文件` 表示“本轮明确附带该文档”；`@文件夹/#标签` 表示“本轮仅在该范围内检索”，两者不会混淆。
- 为保证 Markdown 权威性和引用可审计性，未保存编辑器选区不自动写盘，也不发送客户端拼接正文；用户保存后即可精确引用。
- 不新增第三方依赖；不修改用户 Markdown 内容。
